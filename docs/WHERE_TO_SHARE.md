# Dissemination options

Primary source of truth: the public repository  
https://github.com/coldbricks/charm13

## Recommended path

1. Maintain code, figures, and certificates on GitHub.  
2. Tag releases (e.g. `v0.3.5`).  
3. Deposit a frozen snapshot on **[Zenodo](https://zenodo.org/)** for a DOI (optional GitHub–Zenodo integration).  
4. Optionally write a short technical note (problem, definitions, theorems, limitations, product mapping) before any preprint server submission.

## Preprint and archive venues

| Venue | Role | Notes |
|-------|------|--------|
| **Zenodo** | Open archive + DOI | Low barrier; suitable for software + research notes; deposits are durable |
| **OSF** | Project workspace + preprints | Good for organizing paper, code, and artifacts together |
| **arXiv** | Disciplinary preprint server | Requires appropriate category framing and often endorsement for new submitters; reject overclaim |
| **HAL** | Open archive | Strong complement or alternative to arXiv |
| **GitHub Releases** | Versioned source tarballs | Standard for software; pair with Zenodo for DOI |

## Framing for technical audiences

Prefer evaluation / detection language:

> Finite-model adaptive inspection of attributed filesystem habitats; engineering system for cover construction with refuse-on-detection.

Avoid framing that implies cryptographic breakthroughs, laboratory evasion, or unbreakable camouflage.

## Venues to treat cautiously

| Venue | Caution |
|-------|---------|
| Unsolicited “journals” | Often predatory |
| viXra as first choice | Open but low signal-to-noise for mathematics |
| Social academic networks alone | Useful for links, not a substitute for a DOI archive |

## What dissemination is not

A DOI or GitHub star count is not peer review. The repository already records novelty packaging as application of known sequential-testing ideas with exact synthetic envelopes—not a claim of new core probability theory.
